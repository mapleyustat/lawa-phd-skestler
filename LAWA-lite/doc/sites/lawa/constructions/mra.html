==title:The root class for all MRAs== The root class for all MRA (multi resolution analysis) types used in [LAWA]. This class is just empty since all implemented types of MRAs are specializations of it. The distinction between different MRA types is realized by template parameters.

==section: Template parameters== The declaration of [MRA] reads ==code.cc== template <typename T,

FunctionSide Side, DomainType Domain, Construction Cons>

stru

with template parameters ==list:begin==

->[T]:

the (numerical) element type (e.g. double)

->[FunctionSide]:

the kind of B-spline ([Primal] or [Dual])

->[Domain]:

the domain (e.g. [R], [Interval])

->[Cons]:

the underlying construction.

==list:end==
